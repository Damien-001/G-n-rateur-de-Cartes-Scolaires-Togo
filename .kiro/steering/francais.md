# Règle de Communication en Français

## Directive Principale

Lorsque l'utilisateur communique en français, tu DOIS répondre en français de manière naturelle et fluide.

## Règles de Communication

1. **Détection de la langue**: Si l'utilisateur écrit en français, réponds en français
2. **Ton naturel**: Utilise un français conversationnel et professionnel
3. **Terminologie technique**: 
   - Utilise les termes techniques en anglais quand c'est la norme (ex: "commit", "pull request", "build")
   - Traduis les concepts généraux en français
4. **Code et documentation**: 
   - Les commentaires de code peuvent être en français
   - Les noms de variables/fonctions restent en anglais (convention de programmation)
   - Les messages d'erreur utilisateur doivent être en français
5. **Clarté**: Privilégie toujours la clarté, même si cela nécessite d'utiliser un terme anglais entre parenthèses

## Exemples

**Bon** ✅:
- "J'ai créé le composant CameraCapture avec les props nécessaires"
- "Le build a réussi sans erreurs TypeScript"
- "Voici le résumé des tâches complétées"

**À éviter** ❌:
- Mélanger français et anglais de manière incohérente
- Traduire littéralement des termes techniques qui n'ont pas d'équivalent français standard

## Application

Cette règle s'applique à:
- Toutes les réponses conversationnelles
- Les résumés et rapports
- Les explications techniques
- Les messages de confirmation

Cette règle ne s'applique PAS à:
- Le code source (reste en anglais)
- Les noms de fichiers et chemins
- Les commandes shell
- Les identifiants techniques
