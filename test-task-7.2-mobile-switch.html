Test Task 7.2 - Mobile Camera Switch Button
Ce test vérifie que le bouton de changement de caméra s'affiche UNIQUEMENT sur les appareils mobiles et qu'il est désactivé pendant l'initialisation de la caméra.

📋 Objectif du Test

Vérifier que l'implémentation de la tâche 7.2 respecte les exigences suivantes :

  • Requirement 9.4: Le bouton de changement de caméra doit être affiché uniquement sur mobile
  • Requirement 11.2: Le bouton doit utiliser l'icône RotateCw de Lucide React
  • Le bouton doit être désactivé pendant l'initialisation de la caméra
  • Le bouton doit être positionné au-dessus de la prévisualisation vidéo

🖥️ Test 1: Desktop - Le bouton NE doit PAS s'afficher

  1. Ouvrir l'application sur un navigateur desktop (Chrome, Firefox, Edge, Safari)
  2. Naviguer vers le formulaire d'ajout d'élève
  3. Cliquer sur "Prendre une photo" pour activer le mode caméra
  4. Autoriser l'accès à la caméra si demandé
  5. Observer la prévisualisation vidéo

✅ Résultat Attendu (Desktop)

  • La prévisualisation vidéo s'affiche correctement
  • Le bouton de changement de caméra (RotateCw) NE doit PAS être visible
  • Seul le cadre de guidage blanc doit être visible sur la vidéo
  • Le bouton "Capturer" doit être visible en bas

📱 Test 2: Mobile - Le bouton DOIT s'afficher

  1. Ouvrir l'application sur un appareil mobile (iPhone Safari, Android Chrome)
  2. Naviguer vers le formulaire d'ajout d'élève
  3. Cliquer sur "Prendre une photo" pour activer le mode caméra
  4. Autoriser l'accès à la caméra si demandé
  5. Observer la prévisualisation vidéo

✅ Résultat Attendu (Mobile)

  • La prévisualisation vidéo s'affiche correctement
  • Le bouton de changement de caméra (RotateCw) DOIT être visible en haut à droite
  • Le bouton doit avoir un fond semi-transparent noir (bg-black/50)
  • Le bouton doit être rond avec l'icône RotateCw blanche
  • Le bouton doit être positionné au-dessus de la vidéo (position absolute)

⏳ Test 3: État Désactivé Pendant l'Initialisation

  1. Sur mobile, activer le mode caméra
  2. Observer le bouton de changement de caméra pendant l'initialisation
  3. Essayer de cliquer sur le bouton pendant l'initialisation
  4. Attendre que la caméra soit complètement initialisée
  5. Essayer de cliquer sur le bouton après l'initialisation

✅ Résultat Attendu (État Désactivé)

  • Pendant l'initialisation (isInitializing=true):
    • Le bouton doit être désactivé (disabled)
    • Le bouton doit avoir une opacité réduite (opacity-50)
    • Le curseur doit indiquer "not-allowed"
    • Cliquer sur le bouton ne doit rien faire
  • Après l'initialisation (isInitializing=false):
    • Le bouton doit être actif
    • Le bouton doit avoir une opacité normale
    • Cliquer sur le bouton doit changer de caméra

🔄 Test 4: Fonctionnalité de Changement de Caméra

  1. Sur mobile, avec la caméra active
  2. Cliquer sur le bouton de changement de caméra
  3. Observer le changement de caméra (avant → arrière ou arrière → avant)
  4. Vérifier que la nouvelle caméra s'initialise correctement
  5. Cliquer à nouveau pour revenir à la caméra précédente

✅ Résultat Attendu (Changement de Caméra)

  • La caméra actuelle doit s'arrêter
  • Un court délai (100ms) doit se produire
  • La nouvelle caméra doit s'initialiser
  • La prévisualisation doit afficher le flux de la nouvelle caméra
  • Le bouton doit rester fonctionnel pour changer à nouveau

🔍 Détection Mobile - Implémentation

La détection mobile utilise deux méthodes combinées :

// 1. Media Query pour la taille d'écran const mobileMediaQuery = window.matchMedia('(max-width: 768px)'); // 2. User Agent pour les appareils mobiles const mobileUserAgent = /Android|webOS|iPhone|iPad|iPod|BlackBerry|IEMobile|Opera Mini/i.test(navigator.userAgent); // Le bouton s'affiche si l'une des conditions est vraie setIsMobile(mobileMediaQuery.matches || mobileUserAgent);

⚠️ Points d'Attention

  • Responsive Design: Tester avec différentes tailles d'écran (redimensionner la fenêtre)
  • Tablettes: Les tablettes peuvent être détectées comme mobiles selon leur taille d'écran
  • Mode Développeur: Utiliser les outils de développement pour simuler un appareil mobile
  • User Agent: Certains navigateurs permettent de modifier le user agent

📝 Checklist de Validation

  • Desktop Chrome: Bouton de changement de caméra NON visible
  • Desktop Firefox: Bouton de changement de caméra NON visible
  • Desktop Edge: Bouton de changement de caméra NON visible
  • Desktop Safari: Bouton de changement de caméra NON visible
  • Mobile Safari iOS: Bouton de changement de caméra VISIBLE
  • Mobile Chrome Android: Bouton de changement de caméra VISIBLE
  • Bouton désactivé pendant l'initialisation (mobile)
  • Bouton actif après l'initialisation (mobile)
  • Changement de caméra fonctionne correctement (mobile)
  • Icône RotateCw utilisée pour le bouton
  • Bouton positionné en haut à droite de la vidéo
  • Bouton a un fond semi-transparent noir

Ouvrir l'Application
