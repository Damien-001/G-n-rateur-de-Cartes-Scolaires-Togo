🎥 Test Tâche 8.4 - Camera Lifecycle Cleanup

Vérification de l'arrêt de la caméra lors des événements du cycle de vie du formulaire

📋 Requirements Validés

✅ Requirement 8.1: Stop camera on form cancel (onCancel callback)
✅ Requirement 8.2: Stop camera on form submit (handleSubmit)
✅ Requirement 8.4: Stop camera on component unmount (useEffect cleanup)
ℹ️ Objectif: S'assurer que la caméra est arrêtée automatiquement lors de tous les événements du cycle de vie du formulaire StudentForm pour préserver la batterie et la confidentialité.
⚠️ Important: Pour ces tests, vous devez observer le voyant de la caméra de votre appareil (LED sur ordinateur portable, icône dans la barre de statut sur mobile). Le voyant doit s'éteindre lorsque la caméra est arrêtée.

Test 1: Arrêt de la caméra lors de l'annulation

1. Ouvrir l'application et cliquer sur "Ajouter un élève"
2. Dans la section "Photo de l'élève", cliquer sur "Prendre une photo"
3. Autoriser l'accès à la caméra si demandé
4. Vérifier que le flux vidéo est visible et que le voyant de la caméra est allumé
5. Cliquer sur le bouton "Annuler" (en bas du formulaire)
✅ Résultat attendu: Le voyant de la caméra doit s'éteindre immédiatement et le formulaire doit se fermer.

Test 2: Arrêt de la caméra avec le bouton X (mobile)

1. Ouvrir l'application sur mobile ou réduire la fenêtre du navigateur
2. Cliquer sur "Ajouter un élève"
3. Cliquer sur "Prendre une photo"
4. Vérifier que la caméra est active
5. Cliquer sur le bouton "X" dans l'en-tête du formulaire
✅ Résultat attendu: Le voyant de la caméra doit s'éteindre et le formulaire doit se fermer.

Test 3: Arrêt de la caméra avec le bouton X (desktop preview)

1. Ouvrir l'application sur desktop (écran large)
2. Cliquer sur "Ajouter un élève"
3. Cliquer sur "Prendre une photo"
4. Vérifier que la caméra est active
5. Cliquer sur le bouton "X" dans la section preview (à droite)
✅ Résultat attendu: Le voyant de la caméra doit s'éteindre et le formulaire doit se fermer.

Test 4: Arrêt de la caméra lors de la soumission du formulaire

1. Ouvrir l'application et cliquer sur "Ajouter un élève"
2. Cliquer sur "Prendre une photo"
3. Capturer une photo et cliquer sur "Valider"
4. Remplir tous les champs obligatoires du formulaire
5. Cliquer sur "Ajouter" pour soumettre le formulaire
✅ Résultat attendu: Le formulaire doit être soumis avec succès et le voyant de la caméra doit rester éteint (la caméra a été arrêtée lors de la validation de la photo).

Test 5: Arrêt de la caméra au démontage du composant

1. Ouvrir l'application et cliquer sur "Ajouter un élève"
2. Cliquer sur "Prendre une photo"
3. Vérifier que la caméra est active
4. Ouvrir les DevTools (F12) et aller dans l'onglet Console
5. Fermer l'onglet du navigateur ou naviguer vers une autre page
✅ Résultat attendu: Le voyant de la caméra doit s'éteindre automatiquement lors de la fermeture/navigation. Aucun warning ne doit apparaître dans la console.

📝 Détails d'Implémentation

1. useEffect Cleanup (Requirement 8.4)

// Cleanup camera on component unmount (Requirement 8.4) useEffect(() => { return () => { cameraCaptureRef.current?.stopCamera(); }; }, []);

2. handleCancel Wrapper (Requirement 8.1)

// Handle form cancel with camera cleanup (Requirement 8.1) const handleCancel = () => { cameraCaptureRef.current?.stopCamera(); onCancel(); };

3. handleSubmit Modification (Requirement 8.2)

await onSubmit(studentData); // Stop camera after successful form submission (Requirement 8.2) cameraCaptureRef.current?.stopCamera();
