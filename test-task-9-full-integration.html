✅ Task 9 - Full Integration Test

Checkpoint - Ensure full integration works between CameraCapture and StudentForm

📋 Test Overview

This checkpoint verifies that the complete integration between CameraCapture and StudentForm works correctly. All previous tasks (1-8) have been completed successfully. This test ensures that all features work together seamlessly in a real-world scenario.

🎯 Test Objectives

Verify that users can successfully capture photos using the camera, switch between upload and camera modes, and that all lifecycle events (form submit, cancel, mode switching) properly clean up camera resources.

1 Mode Selector Tests

  • Display both mode options: Verify that "Télécharger une photo" and "Prendre une photo" buttons are visible UI
  • Visual feedback: Selected mode button should have emerald background and shadow UI
  • Mode switching: Click between modes - should switch without losing form data critical
  • Camera cleanup on switch: When switching from camera to upload mode, camera should stop (check browser indicator) critical

2 Camera Initialization Tests

  • Permission request: Browser should prompt for camera permission when selecting camera mode critical
  • Loading state: "Initialisation de la caméra..." message with spinner should appear UI
  • Video preview: Live camera feed should appear in 3:4 aspect ratio (300x400px minimum) critical
  • Frame overlay: White border guide should be visible over video preview UI
  • Initialization time: Camera should initialize within 2 seconds on desktop, 3 seconds on mobile desktop mobile

3 Photo Capture Tests

  • Capture button: "Capturer" button with camera icon should be visible while video is active UI
  • Capture action: Click "Capturer" - should capture current video frame critical
  • Capture speed: Photo capture and processing should complete within 500ms critical
  • Camera stops: After capture, video stream should stop (browser camera indicator should turn off) critical

4 Photo Preview and Validation Tests

  • Preview display: Captured photo should appear in same 3:4 aspect ratio as video preview UI
  • Action buttons: "Reprendre" and "Valider" buttons should be visible UI
  • Retake functionality: Click "Reprendre" - should discard photo and restart camera critical
  • Confirm functionality: Click "Valider" - photo should appear in form and live preview card critical
  • Mode switch after confirm: After validation, should automatically switch back to upload mode UI

5 Mobile-Specific Tests mobile only

  • Camera switch button: RotateCw button should appear in top-right corner of video preview mobile
  • Default camera: Should start with front-facing camera (facingMode: 'user') mobile
  • Camera switching: Click switch button - should toggle to rear camera and back mobile
  • Responsive layout: Video preview should adapt to mobile screen width mobile
  • Mobile performance: Camera initialization should complete within 3 seconds mobile

⚠️ Mobile Testing Required

Test on actual mobile devices (Safari iOS 14.5+, Chrome Android 90+) to verify camera switching and responsive layout. Desktop browsers typically don't have camera switching functionality.

6 Error Handling Tests

  • Permission denied: Deny camera permission - should show error message and fallback to upload button critical
  • Error message clarity: Error message should explain how to enable camera access in browser settings UI
  • Fallback button: "Utiliser l'upload de fichier" button should switch to upload mode UI
  • Camera in use: If camera is used by another app, should show appropriate error message critical

7 Resource Cleanup Tests critical

  • Cleanup on photo confirm: After clicking "Valider", camera indicator should turn off critical
  • Cleanup on mode switch: When switching from camera to upload mode, camera should stop critical
  • Cleanup on form cancel: Click "Annuler" button - camera should stop immediately critical
  • Cleanup on form submit: Submit form with captured photo - camera should stop critical
  • No memory leaks: Capture multiple photos in succession - no performance degradation critical

🔍 How to Verify Camera Cleanup

Check your browser's camera indicator (usually in the address bar or system tray). The indicator should turn off when the camera is properly stopped. On Chrome, look for the camera icon in the address bar. On Safari, check the menu bar for the camera indicator.

8 Integration with StudentForm Tests

  • Photo in formData: Captured photo should populate formData.photoUrl field critical
  • Live preview card: Captured photo should appear in IDCard preview on the right side critical
  • Photo deletion: X button on photo thumbnail should work with captured photos UI
  • Form validation: Form should submit successfully with captured photo critical
  • Photo format consistency: Captured photo should have same format as uploaded photos (300x400px, JPEG 85%) critical

9 Cross-Browser Compatibility Tests

  • Chrome Desktop: Camera access, capture, and cleanup work correctly desktop
  • Firefox Desktop: Camera access, capture, and cleanup work correctly desktop
  • Edge Desktop: Camera access, capture, and cleanup work correctly desktop
  • Safari iOS 14.5+: Front camera, capture, cleanup, and camera switching work mobile
  • Chrome Android 90+: Front camera, capture, cleanup, and camera switching work mobile

🌐 Browser Testing Notes

Desktop browsers typically only have one camera (webcam), so camera switching won't be available. Mobile browsers (Safari iOS, Chrome Android) support camera switching between front and rear cameras.

10 End-to-End User Flow Tests

  • Complete flow: Open form → Select camera mode → Grant permission → Capture photo → Validate → Submit form critical
  • Retake flow: Capture photo → Click "Reprendre" → Capture again → Validate → Submit critical
  • Mode switching flow: Start with upload → Switch to camera → Capture → Switch back to upload → Upload file critical
  • Cancel flow: Select camera mode → Grant permission → Click "Annuler" → Camera stops critical
  • Error recovery flow: Deny permission → See error → Click fallback → Use upload mode critical
